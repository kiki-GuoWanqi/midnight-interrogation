# 《午夜审讯》游戏设计文档

> 面向 Claude Code 的完整开发指南

---

## 一、项目概述

**游戏类型**：AI驱动的文字推理游戏  
**运行环境**：Claude Artifact（React，内置API，用户无需任何配置）  
**目标用户**：面试官/评审者，打开链接即玩  
**核心体验**：玩家扮演侦探，通过自由对话审讯3个AI嫌疑人，找出凶手  

---

## 二、核心游戏循环

```
游戏开始
  ↓
【随机生成案件】从预设案件库随机抽取一个案件，凶手在3个嫌疑人中随机指定
  ↓
【案情简报】展示案件摘要、受害者信息、已知线索
  ↓
【自由审讯】玩家选择任意嫌疑人进行对话，可随时切换，每人最多15问
  ↓
【收集线索】玩家手动标记对话中的关键信息，保存到线索板
  ↓
【最终指控】选择凶手 + 填写指控理由（1-3句话）
  ↓
【AI裁判】将玩家的指控理由 vs 案件真相，交给Claude判断逻辑是否成立
  ↓
【结局揭晓】公布真凶、完整案件经过、玩家推理评分
```

---

## 三、重玩性设计

采用**"固定角色 × 随机凶手"**方案：

- 预设 **3套案件模板**，每套有固定的3个嫌疑人和案件背景
- 每局开始时，从3人中**随机指定一人为凶手**
- 系统根据当局凶手身份，动态生成每个人的 System Prompt（谎言、动机、时间线随凶手身份变化）
- 玩家每次开局都是新局面，即使同一案件也不能靠记忆取胜

---

## 四、案件模板库（3套）

### 案件A：《顶楼的红酒》
- **背景**：科技公司CEO在顶层公寓被毒杀，氰化物溶于红酒
- **嫌疑人**：妻子 / 合伙人 / 秘书
- **核心矛盾点**：不在场证明的时间漏洞、遗嘱修改、情感纠葛

### 案件B：《画廊深夜》
- **背景**：知名画家在私人画廊陈尸，头部钝器伤
- **嫌疑人**：经纪人 / 前助理 / 收藏家
- **核心矛盾点**：赝品交易、版权纠纷、失踪的展品

### 案件C：《别墅除夕》
- **背景**：家族掌门人在家族聚会当晚坠楼身亡，现场疑点重重
- **嫌疑人**：长子 / 继女 / 家庭医生
- **核心矛盾点**：遗产争夺、药物处方、监控盲区

## 五、AI实现方案

### 嫌疑人对话
每个嫌疑人 = 独立的 Claude API 调用，System Prompt 由代码动态生成，包含：

```
- 角色身份、年龄、性格特征
- 与受害者的关系和动机
- 案发当晚完整时间线（含隐瞒的部分）
- 本局是否为凶手（决定谎言深度和慌乱触发词）
- 与其他两个嫌疑人的关联信息（可相互印证或矛盾）
- 行为规则：
    · 不主动暴露关键信息，需被追问才透露
    · 被问到敏感词时，在回复开头加标记 [紧张] 或 [慌乱]
    · 无辜者也有自己的秘密，不能表现得过于配合
    · 角色始终保持一致，不因多轮对话而崩人设
```

每次 API 调用携带该嫌疑人的**完整对话历史**，保证前后一致性。

### 最终裁判
玩家提交指控后，调用一次 Claude API：
- 输入：玩家的指控理由 + 案件真实信息
- 输出：JSON格式，包含 `correct`(boolean)、`score`(1-100)、`comment`(评语)、`truth`(完整真相)

## 七、视觉风格

**黑色电影 Noir 风**，有强烈的年代感和悬疑氛围：

| 元素 | 规格 |
|------|------|
| 主背景 | 近黑深色 `#0a0a0f` |
| 卡片/面板 | 深灰 `#111118` |
| 主强调色 | 琥珀金 `#c9a84c` |
| 危险/指控 | 暗红 `#cc3333` |
| 正文字色 | 暖白 `#e8e0d0` |
| 标题字体 | Playfair Display（衬线，古典感）|
| 正文字体 | Crimson Text（可读性强的衬线体）|
| 特效字体 | Courier Prime（档案/打字机感）|
| 纹理 | 轻微纸张噪点纹理覆盖，增加质感 |
| 动效 | 打字机逐字出现效果（嫌疑人回复）|
| 嫌疑人头像 | 高对比度黑白风格卡片 |
| 状态指示 | 嫌疑人卡片有情绪状态标签：平静/紧张/慌乱 |

## 八、语音系统

使用浏览器原生 **Web Speech API**，零配置，Artifact 直接可用，支持中文。

### 每个嫌疑人的声线配置

| 嫌疑人角色 | pitch | rate | 效果 |
|-----------|-------|------|------|
| 中年男性（合伙人/长子/经纪人） | 0.85 | 0.9 | 低沉沧桑 |
| 中年女性（妻子/继女） | 1.1 | 0.95 | 沉稳克制 |
| 年轻女性（秘书/前助理） | 1.3 | 1.0 | 清脆带紧张感 |
| 老年男性（医生/收藏家） | 0.75 | 0.85 | 苍老迟缓 |

### 触发规则

- 嫌疑人回复文字**逐字打出的同时**，语音同步播放
- 播放时嫌疑人头像有轻微"口型"动画（缩放或波形）
- 情绪状态影响语速：`[紧张]` → rate +0.1，`[慌乱]` → rate +0.2、pitch 微升
- 右上角常驻**静音按钮**，玩家可随时关闭
- 切换嫌疑人或玩家开始输入时，立即中断当前语音（`speechSynthesis.cancel()`）

### 实现要点

```javascript
const speak = (text, suspectVoiceConfig) => {
  speechSynthesis.cancel()
  const utter = new SpeechSynthesisUtterance(text)
  utter.lang = 'zh-CN'
  utter.pitch = suspectVoiceConfig.pitch
  utter.rate = suspectVoiceConfig.rate
  const voices = speechSynthesis.getVoices()
  const zhVoice = voices.find(v => v.lang.includes('zh'))
  if (zhVoice) utter.voice = zhVoice
  speechSynthesis.speak(utter)
}
```

> **注意**：部分浏览器需用户交互后才能触发语音，在首次点击"进入审讯"时做一次静默预热调用即可解决。

## 九、关键游戏状态

```javascript
gameState = {
  currentCase,        // 当前案件模板
  culpritId,          // 本局凶手（随机指定）
  phase,              // 'briefing' | 'interrogation' | 'accusation' | 'reveal'
  suspects: {
    A: { chatHistory: [], questionsLeft: 15, emotionState: 'calm' },
    B: { chatHistory: [], questionsLeft: 15, emotionState: 'calm' },
    C: { chatHistory: [], questionsLeft: 15, emotionState: 'calm' },
  },
  clues: [],          // 玩家手动保存的线索列表
  activeSuspect,      // 当前审讯的嫌疑人
  accusation: {
    targetId: null,
    reason: ''
  }
}
```
